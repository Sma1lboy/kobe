0.7.18 跨 session 体验统一，长跑不再漏内存 06-10 19:29 · 6 commits
  • PERF
    Ops pane 内存泄漏根治

    挂一两天的 ops 进程之前能涨到 2.9GB：file tree 每次 fs-watch 刷新都把所有行推倒重建，opentui 每个 renderable 的生命周期都残留一点 native 内存。现在 git 输出没变的刷新直接跳过，变了的按行身份做增量 reconcile，只重画真正变化的行。

  • FIX
    布局尺寸切任务不再重置

    侧栏宽度、右栏宽度、file-tree/terminal 分割现在是全局共享的一份尺寸：在任何任务里拖好，切到别的任务（包括新建）都长一样，且跟着 tmux server 活 —— 退出 kobe 再开还在。

  • FEAT
    Tasks pane 偏好全局同步

    排序切换（t）和键位图例折叠（?）升级为全局偏好：按一下，所有打开 session 的 Tasks pane 同时变，并持久化到 state.json，重启 kobe 也保留。

  • FEAT
    keybindings.yaml 改了立即生效

    daemon 盯着 ~/.kobe/settings/keybindings.yaml，一改就通知所有 pane 从干净状态重读重应用 —— 删掉一条 override 会正确回到默认键，键位图例同步刷新。（tmux 层的 session 按键仍需 rebuild。）

  • FIX
    F1 帮助全窗口 + 弹窗不再透明

    F1 帮助从挤在窄侧栏里的 overlay 变成独立的全窗口 tab，键位表终于看得全；透明背景模式下 settings、确认框这些弹窗卡片保持完全不透明，文字不再和底下的终端内容糊在一起。

0.7.19 长会话更轻快，大仓库任务不再假死 06-10 20:55 · 7 commits
  • FIX
    第二轮内存泄漏大扫除

    又修掉 5 处长会话泄漏：sidebar 切 task 不再重建所有行的渲染对象、已删除 task 的引擎状态会被清理、断线重连不再留下孤儿循环和永远挂着的 RPC、自动标题也不再因为 substring 把几 MB 的消息 buffer 钉在内存里。

  • FEAT
    导航键和循环键也能重绑了

    之前固定死的方向类按键 —— 列表/文件树上下移动、折叠展开、视图和 tab 前后切换 —— 现在都能在 keybindings.yaml 里改成自己顺手的键（比如 w/s 上下）。配置写错会自动保留默认。

  • FEAT
    大仓库建 worktree 有实时进度

    在很大的仓库上新开 task，git worktree add 可能要跑一分钟，以前任务行毫无反应像卡死。现在 daemon 通过 task.jobs 频道广播进度，所有 Tasks 面板都能看到该行转圈显示 materializing，跑完或失败都保证有终态。

  • PERF
    改动统计改由 daemon 统一采集

    侧栏每个任务的 +N −M 小标签，以前是每个面板各自跑 git status —— N 个 session 看 M 个 task 就是 N×M 个重复进程。现在 daemon 只跑一份并推送给所有面板，连着 daemon 时面板端零 git 进程（daemon 不可达时回退本地轮询）。

0.7.20 闲置效率大扫除，挂机的 kobe 几乎零开销 06-10 22:51 · 7 commits
  • PERF
    切任务明显更跟手

    每次切换任务都要确认 tmux session 是否健康，这条热路径从 10 次 tmux 子进程调用压到 4 次。开十几个任务来回跳的时候，切换延迟肉眼可感地变小。

  • PERF
    闲置时不再狂读 transcript

    判断引擎一轮是否回完，之前每 1.5 秒把几 MB 的 JSONL transcript 整个重读一遍；现在按文件 mtime 缓存，内容没变就一次都不读。挂着一堆长会话过夜，磁盘 IO 基本归零。

  • PERF
    后台轮询全面瘦身

    ChatTab 自动命名从每分钟约 450 次 tmux 调用降到约 165 次；侧栏 branch 标签改成直接 stat .git/HEAD，不再每 2 秒 spawn 一个 git 进程；空闲 spinner 不再每秒 10 次重建所有任务行。kobe 开着不动时，不再偷偷吃你的机器。

  • PERF
    daemon 推送与按键更省

    daemon 向所有订阅 pane 广播时每帧只序列化一次，而不是每个订阅者各一遍 —— pane 越多省得越多；键位查表改成 O(1)，按键不再线性扫整张 keymap。

  • FEAT
    pane 聚焦不再穿边环绕

    ctrl+h/j/k/l 方向聚焦到了窗口边缘就停下，最左侧再按 ctrl+h 不会瞬移到最右边（zoom 状态下也不失灵）；Tasks pane 里按 → 可直接跳回当前窗口的 engine pane。

— 该版本没有此类改动 —

0.7.22 补齐脚本善后，daemon 与 web 面板双线降耗提速 06-11 00:23 · 10 commits
  • FIX
    脚本删任务不再留孤儿引擎

    之前用 kobe api delete/archive 删掉或归档任务，记录没了，但 tmux 里的 engine 子进程还在后台跑 —— 所有 UI 都看不见它，白烧资源，只能靠 kobe reset 救。现在 CLI 照 TUI 的规矩把 session 杀干净：删除必杀，归档时也杀。

  • FIX
    原地切引擎后状态不再串台

    Tasks pane 按 v 换引擎再进任务，以前 turn 状态、● new 活动徽章和 tab 自动命名会一直读旧引擎的 transcript。现在会重新绑定新引擎的 session 并重建 Ops pane，遥测全部跟新引擎走；空闲时徽章轮询退避到 20s。

  • FIX
    输入与快捷键小修合集

    New Task 对话框里 Ctrl+A 回行首恢复可用（之前被 Adopt 页签的全选键吞掉）；底部键位图例跟随你的改键显示真实键位；live-session 路径重建的任务也会跑仓库 init script；kobe hook 不再多挂 500ms；引擎二进制探测改为进程内缓存。

  • PERF
    daemon 闲时按需干活

    订阅按 channel 过滤，pane 只收自己要的数据；没有任何 pane 订阅时，git-status 和自动命名 collectors 直接暂停，有人连上才恢复；连接失败的半成品客户端正确释放，不再泄漏重连循环。

  • PERF
    web 面板提速瘦身

    内嵌终端 scrollback 改成有界 chunk ring，不再每个 PTY 输出块都重拼 256KB 大字符串；SSE 桥只转发浏览器用到的 channel；/api/diff 对 untracked 文件改 ≤8 并发的 worker pool，几百个新文件也不卡；npm 包发布前清空旧 web 构建产物，安装包不再越积越大。
