# kobe Brand Studio Brief

kobe is a local-first terminal UI for running many AI coding sessions at once. The product unit is a task: a git worktree, an engine session, and a branch. The marketing work should show the actual terminal-native product instead of inventing a generic AI dashboard.

Primary audience:

- Developers who live in terminals, tmux, SSH, and local devboxes.
- AI coding power users who need isolated parallel sessions.
- Teams evaluating local-first orchestration instead of hosted IDE workflows.

Core promise:

- Parallel isolated worktrees.
- Persistent engine sessions.
- Terminal-native task switching.
- One place to drive many active coding streams.

Visual direction:

- Make the kobe TUI the main subject, using local screenshots as canonical references.
- Prefer warm technical materials: ink, paper, clay, shell, graphite, copper, and one restrained blue signal accent.
- Keep text sparse and legible. The core lockup is `kobe` with `parallel agents, one terminal`.
- Avoid generic AI glow, fake browser dashboards, robots, mascot language, basketball references, official Anthropic marks, and unreadable terminal walls.

The current production campaign is `mars-kobe`: a cinematic Mars field-computing image where kobe is visibly running on a rugged terminal workstation. The Mars environment is campaign-specific and should not override the default clean product-promo style.
