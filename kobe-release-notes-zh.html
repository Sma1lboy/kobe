kobe — 终端里的多任务 AI 编码工作台 · 发版速览

kobe 近期发版

本地优先的终端 UI，一屏同时跑很多个 AI 编码会话。每个任务 = 一个 git worktree + 一个引擎会话 + 一个分支。 以下是 0.7.34 → 0.7.37 这几个版本的用户可见变化。

最新 0.7.37 区间 0.7.34–0.7.37 日期 2026-06-22 → 06-24 包 @sma1lboy/kobe
RELEASES
v0.7.37 2026-06-24 今天
FEAT

Tasks 侧栏实时显示 PR 检查状态 KOB-10

守护进程会对每个任务分支轮询 gh pr view(仅 GitHub),把结果写回任务,侧栏行右侧多出一个状态徽标:✓ 通过 / ✗ 失败 / • 进行中,随 CI 推进实时更新,不用离开 TUI。状态会持久化(随快照下发,网页看板也能看到,守护进程重启后仍在),且只在 gh 调用成功时写入——所以掉网或未登录都不会误清已知状态。无 PR 的分支、已合并/关闭的 PR 会退避轮询,无界面挂载时则完全暂停。

FEAT

新增 Zen 模式,把 ChatTab 收起到只剩引擎面板

从文件列表上方的 zen 标签(在 create PR 左边)触发,或用 tmux prefix+space。它会隐藏文件/Ops 与终端面板;Tasks 栏也会一起隐藏,除非打开新增的「设置 → 通用 → Zen 模式保留 Tasks 面板」开关(默认开,保留的 Tasks 栏便于退出 zen)。再按一次精确还原 zen 隐藏的那些面板,你原本就收起的面板不受影响。Zen 激活时,保留的 Tasks 面板左下角会有一个 ☯ ZEN 徽标提示当前模式。

FIX

终端变大时工作区跟随生长,不再闪烁

之前为贴合进入客户端而把窗口 window-size 固定为 manual,导致终端从小拉大时窗口不自动长大,UI 被「信箱化」卡在旧的小尺寸,直到切换或重开任务。新增 client-resized 钩子(kobe resync-window)在每次终端尺寸变化时把活动窗口重新钉到客户端尺寸并修复侧栏;拖拽时的事件风暴被合并为一次重钉,窗口与侧栏的重排打包成一条 tmux 命令一次性重绘——终端长大不再闪过变形的中间帧。

FIX

提示词送进 tmux 引擎面板不再偶发卡在输入框

仓库的 init-prompt.md 首条消息、kobe api send 与快速任务投递都受影响:pasteAndSubmit 把粘贴和提交回车连写,二者可能合并成一次 tty 读取,引擎把回车当成粘贴内容而非提交(与网页 composer / PTY sidecar 之前修过的是同一个问题)。tmux 投递路径现在同样把回车延后 ~150ms 拆开发送。

FIX

文件面板路径按码点截断,不再切坏非 ASCII 文件名

窄面板下 truncatePathTail 之前按 UTF-16 码元截断,会把代理对从中切开,使以 emoji 或星平面字符结尾的路径尾(如 …my-🎉-feature.ts)渲染成 �。现在改为按码点切分(与 orchestrator/title.ts 一致),字符保持完整;函数移入纯逻辑模块 filetree/rows.ts 并补了单测。

FIX

文件面板 Changes 标签不再显示裸目录

git status --porcelain 会把整个未跟踪目录折叠成一行 dir/,渲染成没有 +/- 统计、也点不开的目录项。现在改用 --untracked-files=all 运行 git status,把未跟踪目录展开成单个文件(与 All 标签一致并同样遵守 .gitignore);并防御性跳过尾斜杠行,目录再也不会作为变更项出现。

v0.7.36 2026-06-23
FEAT

新增 kobe export,无需守护进程导出任务列表

直接在进程内读取 ~/.kobe/tasks.json 并打印到 stdout:默认 JSON、--csv、或 --format table 对齐表格。可以把任务管道给 jq、用表格软件打开、或在终端里扫一眼——补足了需要守护进程且只出 JSON 的 kobe api list。

FIX

修复重启后任务静默退回 Claude

任务索引加载器之前用过时的 claude | codex 校验持久化引擎,导致 Copilot 任务——或任何用户自注册的自定义引擎——每次守护进程重载 tasks.json 都被悄悄降级回 Claude。现在加载会保留任何已记录的引擎(内置或自定义),仅当从未记录过引擎时才回退到 Claude。

FIX

侧栏打磨

PROJECTS 区在项目较少时不再占满整个滚动上限留下大片空白,而是收缩到实际行数(每张卡 2 行),行数超过上限时仍可滚动。视图标签从「Working session」缩短为「Workspace」,不再在侧栏被截断。

FIX

整窗页面不再响应工作区导航快捷键

新建任务、设置、更新、快速任务、帮助等整窗页面,之前 Ctrl+Q、Ctrl+[ / Ctrl+]、Ctrl+T 等会从 tmux 根表触发,把你从填了一半的对话框里拽走。这些窗口现在带 @kobe_surface 标记,相关快捷键在其中变为空操作。

v0.7.35 2026-06-23
FEAT

新建任务对话框改为自上而下的单一键盘流

模式标签(For Existing / New Repo / Adopt)和引擎选择器现在都是真正的焦点停靠点:对话框打开时聚焦在模式行,←/→ 立即切换模式,Tab 依次走 模式 → 引擎 → 仓库 → 分支 → 创建,Create 按钮移到右下角。仓库/克隆目录选择器里选目录(Enter 或点击)会选中它并前进到下一字段,而不是无限下钻;继续输入则可深入浏览。在最后一个字段按 Enter 直接创建任务(不用再点一次 Create)。

FIX

新建任务对话框焦点样式

聚焦字段的标签(repo、engine、from branch 等)显示为 primary + 加粗 + 下划线,未聚焦标签保持灰。活动模式标签与选中引擎保持 ▸ + 加粗 + primary;输入值保持默认色。替换了此前「聚焦变强调色」的方案——那种读起来太跳。

FIX

文件夹不是 git 仓库时,给出更友好、可执行的报错

不再直接抛出 git 的 fatal: not a git repository,新建任务对话框的内联校验和 worktree 创建 toast 都会解释为什么任务需要 git 仓库,并直接给出修复命令 git init && git add -A && git commit -m "init",并说明非 git 文件夹后续会支持。

v0.7.34 2026-06-22
FEAT

网页 HTTP/SSE 流量直接走守护进程

网页与桌面端不再启动独立的 kobe-web 桥接进程,浏览器流量直接经由 kobe 守护进程。守护进程持有网页路由表、RPC 白名单、SSE 快照流、会话/规格路由与可选静态托管;网页开发和桌面端只需启动 Vite 和 Node PTY sidecar。默认包现在也会捆绑已构建的网页面板,并作为 bun run build 的一部分构建。

FEAT

Tasks 面板项目过滤升级为全局 UI 偏好

在任一会话按 Ctrl+P 会同步更新所有会话的项目范围,进入另一任务不再露出该会话过时的本地过滤状态。Tasks 面板还把折叠的按键图例固定在 tmux 状态栏上方一行,并将侧栏溢出拆成独立的 PROJECTS 与 TASKS 两个滚动区,长任务列表不再把项目行挤出视野。

FEAT

整套 tmux 外观跟随 kobe 主题

专用的 -L kobe tmux 服务器现在从当前主题派生底部状态栏/窗口栏、status-left/right 样式、命令提示、复制模式选区、面板选择器配色与面板边框——切换主题会完整重塑 ChatTab 外观,且不触碰用户真实的 tmux 服务器。同时让设置写入共享状态文件后的主题实时传播更可靠(守护进程轮询小巧的 UI prefs 文件作为兜底)。

FEAT

实验性 kobe-desktop 工作区

一个轻量 Electron 外壳:在空闲本地端口段上启动既有的 kobe web 开发栈并在桌面窗口中打开,不改动守护进程、tmux 或网页桥接架构。
