Run parallel coding agents from any terminal.

kobe turns AI coding work into isolated git worktrees and persistent tmux sessions. AI agents are useful one at a time — kobe is for when you want five attempts running at once.

Get started ↗
Runs on macOS & Linux (Windows via WSL). Requires Bun ≥ 1.3.11, tmux, and one engine CLI on your PATH.

// why kobe

One attempt is a coin flip.
Run five and keep the winner.

kobe gives every attempt its own worktree, branch, and persistent session — so parallel agents never step on each other, and a dropped SSH connection never kills the work.

01 — safe parallelism

Every attempt is isolated.

One prompt fans out into separate worktrees and branches. Five tries run at once and never touch each other's files — compare the diffs, keep the good one, archive the rest.

prompt "simplify auth"
↓ fan out into 3 worktrees
try/claude-arunning
try/codex-brunning

02 — persistent by default

Close your laptop. Work keeps running.

Agents live in tmux on the machine where your code is. A dropped SSH connection never kills the run — detach, reattach tomorrow, pick up exactly where you left off.

03 — real environment

Not a sandboxed approximation.

Agents run right next to your real dependencies, services, credentials, and build cache. What passes for the agent passes for you — because it's the same machine.

deps services $PATH
● agent
secrets build cache local DB
04 — terminal-native

Runs where your code already lives.

Your laptop, a devbox, a VPS — anything you can SSH into. No browser, no VNC, no desktop app to babysit. The terminal is the product.

$ ssh devbox
$ cd repo && kobe
no browser · no VNC · no desktop app
