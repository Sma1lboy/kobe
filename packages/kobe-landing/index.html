GitHub –

Run parallel coding agents from any terminal.

kobe turns AI coding work into isolated git worktrees and persistent hosted engine sessions. AI agents are useful one at a time — kobe is for when you want five attempts running at once.

Get started ↗

Runs on macOS & Linux (Windows via WSL). Requires Bun ≥ 1.3.11 and one engine CLI on your PATH.

one task = git worktree + hosted engine session + branch
1.0 · FAN OUT

One prompt becomes five isolated attempts.

Type the task once. kobe spawns each attempt in its own git worktree on its own branch, running whichever engine CLI you point it at — claude, codex, gemini, copilot, or your own command. Parallel agents never touch each other's files.

$ kobe new "simplify auth" --engine claude --fan 3
✓ 3 worktrees · 3 branches · 3 sessions
2.0 · WALK AWAY

Close the laptop. Work keeps running.

Agents run in hosted PTYs on the machine where your code lives — your laptop, a devbox, a VPS, anything you can SSH into. The standalone PTY Host keeps sessions alive when kobe exits, the daemon restarts, or SSH drops.

And it's your real machine: real dependencies, services, credentials, build cache. What passes for the agent passes for you.

$ exit
Connection to devbox closed.
▸ 41 minutes later, from anywhere
$ ssh devbox && kobe
● 3 sessions live · running 41m
  attached — right where you left off
3.0 · KEEP THE WINNER

Compare the diffs. Land the best one.

Reopen kobe and every attempt is where you left it, each on its own branch. Review the diffs side by side, merge the branch you like, archive the rest. One attempt is a coin flip — five is a choice.

$ kobe changes try/claude-c
+118 −204 · 6 files
$ kobe land try/claude-c
✓ try/claude-c merged to main
○ try/claude-a  archived
○ try/codex-b   archived

Spin it up where your code lives.

Your laptop, a devbox, a VPS, or any machine you can SSH into. No browser, no VNC, no desktop app.

$ bun install -g @sma1lboy/kobe
$ ssh devbox && cd repo
$ kobe
3 attempts running. Go do something else.
